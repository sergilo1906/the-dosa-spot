# QA Severities

## Error

Use `error` when the product is clearly broken or misleading.

Examples:

- missing built route
- visible CTA without useful href
- missing basic SEO tags
- broken heading structure
- missing skip link

## Warning

Use `warning` when the product still works but has a meaningful quality or release risk.

Examples:

- legacy canonical hostname
- visible NAP missing a key local detail
- section visible with weak support
- duplicate asset symptoms

## Polish

Use `polish` when the issue is real but non-blocking.

Examples:

- repeated phrase hints
- compact gallery that still works
- FAQ answers that run long

## Manual Review

Manual review is not a failure severity. It is a checklist for things automation cannot verify safely enough yet.

Examples:

- contrast on real devices
- crop quality
- subtle tone consistency
